# тайми — карта на GitHub Pages

Простая страница с Яндекс.Картой: 34 точки из `places_geo.json`, метки строго по `coords`.

## Файлы
- `index.html` — вся логика внутри, ничего больше не нужно
- `places_geo.json` — координаты и подписи (title, address)

## Как выложить

1. Создайте новый репозиторий на GitHub (публичный).
2. Положите в корень `index.html` и `places_geo.json` (оба файла из этой папки).
3. В `index.html` замените `YANDEX_MAPS_API_KEY_HERE` на настоящий ключ:
   - получить ключ: https://developer.tech.yandex.ru/services/3 → «JavaScript API и HTTP Геокодер»
   - ключ бесплатный, но обязателен — без него карта не покажет тайлы
4. В репозитории: Settings → Pages → Source → Deploy from branch → `main` / `root`.
5. Через минуту-две страница будет доступна на `https://<ваш-юзернейм>.github.io/<репозиторий>/`.

## Если хочется проверить локально до пуша на GitHub
Открыть `index.html` напрямую двойным кликом (`file://…`) не получится — браузер
блокирует `fetch('places_geo.json')` для локальных файлов. Нужно поднять
локальный сервер в этой папке, например:

```
python3 -m http.server 8000
```

и открыть `http://localhost:8000`.

## Если координат станет больше
Просто добавляйте объекты в `places_geo.json` в том же формате
(`title`, `address`, `coords: [lat, lon]`) — `index.html` менять не нужно,
метки и кластеры пересчитаются сами.
