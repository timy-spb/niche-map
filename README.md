# тайми — деплой на GitHub Pages

## Что было не так
GitHub Pages всегда отдаёт файл `index.html` из корня репозитория как главную
страницу. В прошлый раз в корне оказались сразу два кандидата: простая
карта (уже названная `index.html`) и полный лендинг (`taimi-landing.html`).
GitHub показал карту — потому что только у неё было правильное имя.
Полный лендинг лежал рядом, но открыть его можно было только по прямой
ссылке `/taimi-landing.html`, что не совсем то же самое, что «сайт открылся».

## Что залить в репозиторий
Из этой папки — **только это**, ничего больше:

```
index.html      ← это и есть полный лендинг (просто переименован)
map/
  index.html    ← отдельная страница с настоящей Яндекс.Картой
  places_geo.json
```

Папки `fonts/`, `p/`, `pp/`, `ppp/`, файлы `build2.py`, `places.json`,
`README.txt` из старой загрузки — это исходники для пересборки лендинга
у меня в чате, репозиторию GitHub Pages они не нужны и только путают
структуру. Их можно удалить из репозитория (сама сборка уже целиком
запечена внутри `index.html` — шрифты и фото там в base64).

## Как залить (замена старой загрузки)
1. В репозитории `niche-map` удалите всё, что сейчас лежит в корне.
2. Загрузите файлы из этой папки, сохранив структуру (`index.html` в
   корень, `map/` — подпапкой).
3. Settings → Pages должен уже быть настроен на `main` / `root` — трогать
   не нужно.
4. Через минуту-две `https://timy-spb.github.io/niche-map/` откроет
   полный лендинг.

## Настоящая карта
В лендинге в карточке карты («спб тайми») теперь есть маленькая ссылка
«реальная карта →» — она ведёт на `map/`, где настоящая Яндекс.Карта
с 34 точками по координатам из `places_geo.json`.

Перед этим не забудьте заменить `YANDEX_MAPS_API_KEY_HERE` внутри
`map/index.html` на настоящий ключ (иначе карта покажет подсказку про
ключ вместо тайлов) — получить ключ:
https://developer.tech.yandex.ru/services/3
